# GoldAxis NFT Explorer

A **full-stack Web3 application** to explore and showcase the **Golden Baboons Mining Club (GBMC)** NFT collection on Ethereum. It ships with its own ERC-721 smart contract (deployable via Hardhat) and a modern NFT explorer built with Next.js that reads the blockchain in real time, connects wallets via MetaMask, and resolves metadata and images stored on IPFS.

## 🎯 What it does

- **Explores an NFT collection** (GBMC, 8,888 pieces) directly from the Ethereum blockchain, with no reliance on centralized APIs.
- **Connects your wallet** (MetaMask) and automatically detects which NFTs from the collection your address owns by scanning the contract on-chain.
- **Reads metadata and images from IPFS** with a *fallback* system across multiple gateways (Cloudflare, dweb.link, ipfs.io) for maximum availability.
- **Shows each NFT in detail**: name, description, attributes/traits, owner, and direct links to OpenSea and Etherscan.
- **Demo mode**: if you don't connect a wallet (or hold no NFTs from the collection), a sample selection is shown so the app is browsable right away.
- **Bilingual**: Spanish and English UI with a toggle (built-in i18n).
- **Includes its own ERC-721 contract** (`PolyNFT`) ready to deploy and mint your own NFTs with IPFS metadata.

## 🏗️ Architecture

The project is split into two layers that communicate through a configuration file generated at deploy time (`frontend/src/app/contract.json`, which holds the **contract address** and its **ABI**).

```
polygon-nft-app/
├── contracts/
│   └── PolyNFT.sol          → ERC-721 contract (OpenZeppelin)
├── scripts/
│   └── deploy.js            → Deploys and exports address + ABI to the frontend
├── hardhat.config.js        → Network (Polygon) and compiler configuration
├── artifacts/ · cache/      → Hardhat compilation output
│
└── frontend/                → Next.js app (App Router)
    └── src/app/
        ├── page.tsx
        ├── layout.tsx
        ├── globals.css      → "gold/aurora" theme + Tailwind v4
        ├── contract.json    → Generated by deploy.js (address + ABI)
        └── components/
            └── NFTApp.tsx    → Main explorer logic
```

### Data flow

1. **Deploy** → `deploy.js` compiles and deploys the contract, then writes `address` + `abi` into `contract.json`.
2. **On-chain read** → the frontend instantiates the contract with `ethers.js` and a public Ethereum `JsonRpcProvider`.
3. **Ownership detection** → when the wallet connects, `balanceOf` is queried and `ownerOf` is scanned to find the user's tokens.
4. **Metadata resolution** → for each token, `tokenURI` is called, the JSON is fetched from IPFS, and the image and attributes are rendered.

## 🛠️ Technologies

### Smart Contract (blockchain layer)
| Technology | Purpose |
|---|---|
| **Solidity 0.8.28** | Contract language (EVM target: Cancun) |
| **OpenZeppelin Contracts v5** | Secure, audited base for `ERC721` + `ERC721URIStorage` |
| **Hardhat** | Compilation, testing, and deployment |
| **ethers.js v6** | Network interaction in scripts |
| **dotenv** | Manages private keys and secrets via `.env` |

The `PolyNFT` contract (symbol `PNFT`) implements:
- `mint(string uri)` — mints an NFT to `msg.sender` with IPFS metadata and returns the `tokenId`.
- `totalMinted()` — total number of minted tokens.
- Per-token URI storage via `ERC721URIStorage`.
- `NFTMinted(address, tokenId, tokenURI)` event for indexing.

### Frontend (application layer)
| Technology | Purpose |
|---|---|
| **Next.js 16** (App Router) | React framework, static export (`output: "export"`) |
| **React 19** | Declarative UI with hooks |
| **TypeScript 5** | Static typing |
| **ethers.js v6** | Contract reads and wallet connection (`BrowserProvider`) |
| **Tailwind CSS v4** | Styling (gold theme + "aurora" background) |
| **lucide-react** | Iconography |
| **IPFS** | Decentralized storage of metadata and images |

> ℹ️ The frontend is configured as a **static site** (`next.config.ts → output: "export"`), which lets you deploy it to Vercel, Netlify, IPFS, or any static file host with no server required.

## 🚀 Getting started

### Requirements
- Node.js 18+
- A wallet with MATIC/ETH for gas (only if you're deploying)
- MetaMask in the browser (to connect a wallet in the frontend)

### 1. Contract (Hardhat)

```bash
# From the project root
npm install

# Set up your credentials
cp .env.example .env
# Edit .env with your PRIVATE_KEY and RPC_URL

# Compile
npx hardhat compile

# Deploy (network configured in hardhat.config.js)
npx hardhat run scripts/deploy.js --network polygon
```

On deploy, the address and ABI are automatically written to `frontend/src/app/contract.json`, leaving the frontend ready to point at your contract.

### 2. Frontend (Next.js)

```bash
cd frontend
npm install

# Development server
npm run dev
# → http://localhost:3000

# Static production build
npm run build   # generates the out/ folder
```

## ⚙️ Configuration

### Environment variables (root, `.env`)
```env
PRIVATE_KEY=your_private_key_here    # key of the deploying wallet
RPC_URL=https://rpc-amoy.polygon.technology
```

> ⚠️ **Never** commit your `.env` or your private key to a public repository.

### Collection and network (frontend)
The collection, IPFS gateways, and RPC are configured at the top of [`NFTApp.tsx`](frontend/src/app/components/NFTApp.tsx):

```ts
const ETH_RPC = "https://ethereum-rpc.publicnode.com";
const IPFS_GATEWAYS = [ /* cloudflare, dweb, ipfs.io */ ];
const COLLECTION = { name: "Golden Baboons Mining Club", symbol: "GBMC", ... };
```

## 📦 Frontend deployment

Since it's a static export, simply serve the `frontend/out/` folder:

- **Vercel** — detects Next.js automatically.
- **Netlify / Cloudflare Pages** — publish directory: `frontend/out`.
- **IPFS / static hosting** — upload the contents of `out/`.
